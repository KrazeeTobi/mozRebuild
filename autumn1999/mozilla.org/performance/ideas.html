performance: ideas

Contact: Chris Waterson (waterson@netscape.com)
Last Updated: 29-July-1998

This document is a dumping ground for random ideas that people have sent in. These are all relatively "uncooked", but seem reasonable and probably warrant some more investigation. If you want to work on one of them, go for it! I've listed the person that told me about the idea (in the cases where the email address that they used actually worked), so you can contact them for more information or just to discuss it.

    Binary Headers. Use binary headers rather than ASCII to reduce overall transfer sizes.

    Heuristic Content Ordering. Improve heuristics for ordering downloaded content; e.g., place small images before large ones. In addition, possibly allow a content developer to explicitly specify the order in which content gets retrieved. Submitted by Ben Gnomon.

    Cache Preferences By Site. Sometimes you'd like to be able to specify how the cache works on a site-by-site basis. For example, you might want a file: URL to never be cached, or the content from http://www.foobar.com/. This feature would be particularly useful to a content developer: a content developer regularly switches modes between "regular surfing" where the normal cache parameters work just fine, and "development", where the cache gets in the way of the latest changes you just made. Submitted by Magnus Klausen Revang.
