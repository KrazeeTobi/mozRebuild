What are "most frequent bugs"?

The Mozilla QA Most Frequent Bugs page is a living document which lists the known open bugs which occur most frequently in recent builds of mozilla.  This information is provided in order to assist in minimizing the amount of duplicate bugs entered into Bugzilla which in turn cuts down on development time

If you have a "most frequent bug" which occurs to users often and you believe it should be on this list, e-mail the bug # to Jan Leger.

How do I use this list?
  1. Go the product where you found the bug.
  2. Review the most frequent bugs list.  Areas are listed alphabetically for each product.
  3. If problem is listed:
    • Click on Bug # link to confirm that you have found the same bug and comment if you have additional information.  Or move on with your testing of the product.
  4. If problem not listed:
    • Go to the milestone planning page for the product available via link under product title.  Ensure that the problem that you are seeing is a not feature which is not implemented yet.  If it not planned for development until a later milestone, it is not a bug.
    • If feature is in, but not working correctly
      • Go to Bugzilla and post a new bug.
