Class

Definition

  • A class is a plan, or blueprint, for a new object.
  • A class "describes the state space, initial state, and behavior of a new object" (Component Software, p. 31).

Notation

The symbol for a class is the same as specified by UML, namely three rectangles stacked vertically. The top rectangle contains the class name. The middle rectangle may optionally contain a list of properties of objects of that class. The bottom rectangle may optionally contain a list of interfaces or operations implemented by the class. Interfaces should be externally attached to the class on either the left, right, or top edges (see Interfaces section).
