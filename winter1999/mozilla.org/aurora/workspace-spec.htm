Aurora Workspace Spec

Mike Pinkerton, Netscape Communications
Created: 16 July, 1998, with help from John Gable for the original spec
Updated: 31 July 1998

Summary

This document contains the latest proposals for the individual workspaces within Aurora. Please refer to the Aurora Tree View Spec for more information regarding the overall behaviors of an Aurora pane and the Aurora Menus Spec for layout and structure of the menus of Aurora.

Aurora consists of 6 main types of panes:

  • Bookmarks (and any other user created workspace or folder)
  • History
  • Smart Browing
  • Configure Toolbars
The specifics of each pane will be summarized in the remainder of this document, but some things are consistant across all panes:

Drag and Drop

While fundamental to being able to use Aurora for bookmark managment, we have to punt on some issues related to drag and drop for the initial release. Most notably, drags are always moves except when remote data is concerned or the source pane is history. There is currently no modifier key to force a copy.

Context Menus

Since Navigation Mode doesn't have any concept of selection, there aren't any context menus when the tree is in that mode. 

Bookmarks Pane

The Bookmarks Pane should feature most (if not all) of the functionality described in the Pane Spec as it is the primary area in which the user will be organizing their information. The following features must be present in this pane in order for it to be functional as an organizational tool. Note that some of these features come for free from the Aurora Pane, but they are duplicated here for completeness.
  • Rename bookmarks inline
  • Edit the URL of a bookmark
  • Clicking a link opens the correct page in the current window or a new one
  • Delete bookmarks (single, multiple, and folders)
  • Drag and Drop to rearrange the order of items
  • Drag and Drop to manipulate the hierarchy of items
  • Folder creation
  • Separator creation
  • New bookmark creation
  • Import
  • Export (as HTML and RDF, sibling menu commands)
  • Find
  • View multiple columns
  • Sort on a particular column, in particular the "Name" column
  • Marquee selection (to select multiple items at on time) (optional)
  • Cut, Copy, and Paste
  • Undo
This functionality is not specific to the Bookmarks Pane, per se, as any user created workspace (of which the Bookmarks Pane is one) or subfolder placed on the toolbar should behave in this manner. The Bookmarks Pane is special in that it is created at startup with the contents of the user's old Bookmarks.htm[l] file and it cannot be deleted.

By default, the Bookmarks Pane also contains a special folder used by other parts of the chrome. The "Bookmarks Menu Folder" is a container whose contents are displayed in the product-wide "Bookmarks" menu. Initially, this folder is the entire Bookmarks Pane. This special folder cannot be deleted and can be changed to any container in Aurora by context menu. Note that in 4.X, the Personal Toolbar folder was also in the users's bookmarks. In 5.0, this is no longer the case, and the Personal Toolbar folder is now a separate entity configurable from the Configure Toolbars pane.

History Pane

The History Pane is in charge of showing the user their history in a more organized fashion than a long list. Unlike other panes, the user may not edit the items in the History Pane though they may delete them as a way of clearing out certain parts of history (for example, to prevent your co-workers from knowing you routinely visit www.microsoft.com). These are the features which are really necessary:
  • Clicking a link opens the correct page in the current window or a new one
  • Delete items (single, multiple, and folders)
  • Drag items out of pane into other panes/windows
  • Find
  • View multiple columns
  • Sort on a particular column
  • Cut, Copy
  • Undo
  • Marquee selection (optional)
History is organized not by the user, but by Navigator. As a result, drops into the History Pane are not allowed. The pane orgainizes the items in the following way:
  • By Date
    • Within the last hour
    • Today
    • Each day of this week
    • Week of XX/XX/XX
  • By Site
    • a container for each website with all pages in a site inside that container. The hierarchy is flat from this point down no matter how deep the site may go.
  • Most Frequently Visited
    • a container with your 10 most frequently visited pages
  • "Find" results (MAYBE)
    • a container with the results of any "Find" command
Obviously, any given site will appear at least twice (once in "By Date" and once in "By Site"). A site may appear many times in the "By Date" sub-groupings, though not twice within the same sub-grouping. A site will appear only once within the "By Site" grouping regardless of how many times or over what time period the site was visited.

Smart Browsingô Pane

The SmartBrowsing pane, like the Bookmark Pane, should have most of the functionality of the full Aurora Pane. Even though the contents can be modified by the site, the user still has final control over adding, removing, and renaming items. The user needs to be able to, for example, add their own URLs about windsurfing to Yahoo's windsurfing links folder.

The SmartBrowsing Pane also will be the pane which, by default, will make the most use of custom colors, backgrounds, and icons in order to allow the site designer to make the Aurora pane blend seamlessly with their content. Again, the user has the final say over the colors, etc used to display the information in the table. Furthermore, the icon of the sitemap button will change depending on the sitemap being displayed. For example, HotWired uses the Wired logo as the icon for their sitemap pane.

  • Rename bookmarks inline
  • Edit URL of a bookmark
  • Clicking a link opens the correct page
  • Delete items (single, multiple, folders)
  • Drag and Drop to rearrange
  • Drag and Drop to manipulate the hierarchy
  • Folder creation
  • New item creation
  • Export
  • Find
  • View multiple columns
  • Sort on a particular column
  • Marquee selection (optional)
  • Cut, copy, paste
  • Undo
  • Custom colors (icon text, cell colors, table bg color)
  • Custom background (use a GIF)
  • Custom selector icon

Toolbar Configuration

This pane is for manipulating the order and contents of the toolbars. Most importantly, it provides an easy way to replace buttons that may have been deleted. This pane requires, at minimum:
  • Drag and Drop to manipulate the hierarchy
  • Inline editing
  • Undo
  • Cut/copy/paste
  • Delete items (single, multiple, folders)
  • Marquee Selection (optional)
The contents of this pane are containers for each toolbar:
  • Navigation Toolbar
  • Homer Bar
  • Personal Toolbar
Since the personal toolbar is no longer a part of bookmarks, this is where the user can go to edit that bar if they don't want to use drag and drop to manipulate it directly.
