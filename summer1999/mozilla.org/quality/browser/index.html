The Browser and Editor test cases are divided into several different categories of test types. When a new build gets posted, the first type of tests that are used are called Basic Functional tests, (BFTs). These tests verify that the application is functional at a basic level. The Basic Functional Tests are used to touch the product at a high level. If the BFTs pass, then lower level test types are run. These lower-level tests are called Functional Tests.

If you are interested in helping us test, that's great. Just pick the type of testing you are interested in and have fun.

What if you find an error in a test case?
Please email me and let me know about it. Include the test case name and describe in detail what the error is. I will correct the problem as quickly as I can.
What if you find a bug as a result of running a test case?
Well, you'll want to do a couple of things:
  1. Please run a query in the bug database to see if the bug has already been submitted.
  2. When you do submit a bug, please preface the Summary field with one of these:
    [BE:BFT]This will let me know that you found a bug using one of the Browser|Editor Basic Functional tests
    [BE:HTML]This will let me know that the bug was found in one of the HTML Functional tests
What else can you do?
Please include the URI of the test case within the URL field
