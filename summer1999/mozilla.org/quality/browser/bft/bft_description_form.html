Basic Functional Test Case Description - Form

Requirements:

- bft_browser_form.html

Description:

The purpose of this test is to verify that the form objects are placed correctly on the page, and that the information is entered correctly.

  1. Launch Navigator.
  2. Display the FORM element test case.
  3. Enter a value in the password text field.
  4. Select a customer status radio button.
  5. Enter a value for the first name, last name, address, city, state, zip code and telephone number fields.
  6. Enter a quantity, select a product, and enter a sale amount for each line entry.
  7. Enter text in the comment field.
  8. Select the Browse button, and search your disk for any file.
  9. Select the file name from the File Upload dialog box and select Open.
  10. Select the Submit button.

Link(s):

Form test page

Expected Results:

Password field:Should allow up to 19 alpha-numeric values. The value entered should not be visible, but should be displayed as asterisks.
Customer status:The radio buttons should be selectable, and should toggle if each is selected. You should not be able to select both buttons at the same time.
Text entry fields:All text entry fields should allow you to enter text.
  • First name, Last name, Address, and City fields should allow up to 30 alpha-numeric values;
  • State field should allow only two alpha-numeric values;
  • Zip code field should allow up to 15 alpha-numeric values;
  • Telephone field should allow up to 25 alpha-numeric values.
Qty, Product, Sale Amount fields:The Quantity field should allow up to three alpha-numeric values. You should be able to select all of the options in the Product drop-down field. The Sale amount field should allow up to eight alpha-numeric values.
Product info fields:The Quantity field should allow up to three alpha-numeric values; the Product Selection drop-down list should be accessible and selectable, and the Sale Amount field should allow up to eight alpha-numeric values.
Comment field:Should permit you to enter an indefinite number of alpha-numeric values.
Browse selection:This should allow you to enter a file name in the text field or browse your disk and the internet and select a file. After you specify a file, the file name should appear in the associated text field after you close the dialog box.
Submit button:Selecting the Submit button should submit the form.
Reset button:Selecting the Reset button should return all fields and values to the default state.
Tab key:Using the tab key should move the focus from one form element to the next.
