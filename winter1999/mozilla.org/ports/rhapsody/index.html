Where we are now:
  • Autoconf runs the Unix builds now. Things mostly compile once we get autoconf working.
  • XPFE solves a lot of the front-end rearchitecture problem, the port just got A LOT easier.
  • NSPR builds on NeXTSTEP 3.3 with some hacks, 4.2 is proving a little harder
What needs to happen next?
  • Fill in holes in NGLayout backend
    • base/src/unix/nsTimer.cpp: Replace X11 stuff, it's ~ 5 functions.
    • widget/src/rhapsody: This is where the HTML area UI objects like buttons and ListBoxes get defined. We might be able to skip this and just render text to start.
    • gfx: Replace the X11 stuff here.
  • Get webshell/tests/viewer to build & link
  • Get xpfe/xpviewer to build & link
