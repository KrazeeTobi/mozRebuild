documentation

As you are no doubt aware, the Communicator source code is enormous. It is also quite complicated. We're going to be working to provide high-quality technical documentation that helps people find their way around, and hopefully find the pieces of code they're looking for without becoming totally overwhelmed.

This is one of the things that Netscape is busily working on now, with the intent of having at least an initial pass at this documentation available by the time the source is released. It's a big job, and it will also need to grow and evolve as time goes by.

Our current plan for the documentation is something like this, though this could well change in the next few weeks:

    Breadth before depth.
    In general, describe the various components at a gross level, proceding to more detail.

    Identify the components.
    List each module and library, with a general overview of its mission.

    Identify the major data structures and conventions.
    how modules communicate; how the separation between the back-end cross-platform code (``XP'') and the front-end, platform-specific code (``FE'') works; the theory behind the many finite-state machines used in our protocol library; etc.

    Track the flow of control.
    provide a few tutorials that illustrate the interconnetion of the modules. For example:

    • trace the act of clicking on a link, and loading and displaying a web page with forms;
    • trace the display of a page with images and plugins;
    • trace the act of retrieving a message from a POP3 server, saving it, and displaying it in a window;
    • trace the act of hitting the ``reply'' button, composing an HTML message, and sending it to an SMTP server.

Another thing that we hope to bring online at the same time the source becomes available for download is a fully hyperlinked version of the source code, such as that used by the Linux kernel developers: with this tool, you will be able to explore the current version of the source code without first having to download the whole thing.

These plans are all still in flux; what else would you like to see here? What would be most helpful? Let us know.
