by Scott Collins

last modified 12 May 1999

Welcome to the nsCOMPtr page.

nsCOMPtr Documentation

  • nsCOMPtr User Manual describes a `smart-pointer' class for XPCOM.
  • Re: some changes are coming to nsCOMPtr is a news posting describing changes to nsCOMPtr not reflected in the current documentation.
  • getter_doesnt_AddRef is gone (Re: nsCOMPtr questions) is an email message describing getter_AddRefs, the now defunct getter_doesnt_AddRef, and some esoterica about services.
  • getter_AddRefs and friends (Re: nsCOMPtr questions) is an email message that further delves into getter_AddRefs and alternatives.
  • Some COM Ownership Guidelines gives a brief set of guidelines to help you build a supportable ownership model.
  • Code bloat [LONG, summary at top] reports on bloat tests comparing raw pointers, nsCOMPtrs, and `nsIPtr's.
