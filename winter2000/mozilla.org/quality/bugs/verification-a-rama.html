Document Overview
This page provides queries to the hottest Resolved/Fixed bugs needing verification.  This page is most commonly utilized during a "verification-a-rama" where the number of these types of bugs has gotton so large that a posse of folks is needed to bring the numbers down :-)  However, feel free to query these anytime to assist with verficiations.

All the bugs below have been given a "verifyme" keyword.  This keyword is for bugs which are ok verify with latest mozilla build by someone other than the QA Contact indicated.  The important points to remember when taking on one of these bugs:

  • Before Taking the Bug
    • Make sure that you have the correct platform(s) and OS(s) to verify what is outlined in the bug
    • Download the latest daily build for that platform.
    • Always state in the bug that you are taking it to verify.  Comment and Commit. This will keep other folks from working on the same verification and duplicating your efforts.
  • After Verifying the Fix
    • If bug no longer occurs - indicate the build #, platform(s) and OS(s) used to verify the bug in the comments field.   Change the Status field to Verified.
    • If exact bug still occurs - indicate the build #, platform(s) and OS(s) used to verify the bug in the comments field.  Indicate that the bugs still occurs. Change the Status field to Reopened.
    • If the described no longer bug occurs, but you find another problem while verifying -  indicate the build #, platform(s) and OS(s) used to verify the bug in the comments field.   Change the Status field to Verified.  Write a new bug for the new problem.
