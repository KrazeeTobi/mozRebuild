Short term

  • Current Goal: Get a basic HTML window to render simple HTML on the screen.

  • After the MacOS X announcement, the current feeling is that a YellowBox approach makes the most sense.
  • stubfe is close to linking once backend compiles
  • X11 port almost builds (if you build X)

Development Strategy

    Bootstrapping the Application
    • Initial bringup of Navigator:
      • Build Navigator using StubFE.
    • X bringup (Do we want to pursue this?)
      • Get OSF Motif built on Rhapsody + Xnext.
      • Get Navigator to build using X/Motif UI.
      • Get Navigator to run in X.
    • Yellow bringup
      • Model the Yellow UI (Interface Builder)
      • Wrap UI controls with ObjC++.
      • Connect controls to UI model.
      • Get Navigator to run in Yellow.
    Issues
    • Sharing code within the group.
    • Effective code reuse.
      • How to effectly leverage ObjC frameworks without skewing the source base such that keeping in sync is difficult. (This is doable, but takes some planning).
    • Mariner or Raptor layout engine?

Long term

  • Leverage some Rhapsody technology in Comminucator:
    • Loadable bundles as plug-in architecture.
  • Integration of Rhapsody's Java VM into the Communicator sandbox.
  • Create modular frameworks:
    • SGML/HTML (parse+display)
    • Transport (http,ftp,smtp,pop)
    • Image (display)
