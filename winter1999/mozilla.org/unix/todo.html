Making Unix/GTK Go

There's a lot to do in Unix/GTK land, so if you want to lend a hand, you might start by choosing from the following list. ramiro@netscape.com and mcafee@netscape.com are working full-time on this stuff now, so they might be able to help you get started. Or, you could mail all of the GTK module owners.
Performance
Lots of stuff could use performance help, including the event handling and rendering. Also, we're currently creating _way_ too many GtkLayouts. See Matt Wilson's recent post to mozilla.gtk about that issue.
Image scaling
The current image scaling code is a bit wrong, and needs attention.
Fonts
Currently, the font system is brutally hard-coded, and always chooses the same font. Ick.
GFX-on-libart
As we move into the brave new world of rendered-atop-gfx widgets, the performance of our GFX layer will become paramount. Raph Levien has a truly righteous library called libart which would probably be of service.
File a bug
Widgets appear where they're not supposed to? Toolbars disappear when you scroll? Links ignore your fervent attempts to click them? File a bug!
Debug a crash, save a soul
If you can help debug a crash by grovelling through a stack trace and giving background information or even a suggested fix, that's a huge help. File a bug, and include your findings!
