26 Oct 1998:

    New NGLayout strategy:
    • Mozilla has switched over to the new layout engine.
      This means all of the "classic" FE's will need to re-write,
      XFE being one of them. This was a painful decision, but we
      unanimously agree this is the right thing to do for the long run.
    • Brendan's roadmap doc outlines the new NGLayout-based strategy.
    • Until something else shows up, we'll use this XFE page
      to publish status for the Unix/NGLayout implementation.
    What's new for the NGLayout/Unix implementation:
    • Moving from Motif to GTK:
      • Better support from the net, that's YOU
      • Better rendering, font support, color management, etc.
      • GTK is in the public domain
      • Motif implementation can still coexist,
        but we're pushing for a unified GTK effort
    • Moving to autoconf
    • We just got the Linux version of the webshell/tests/viewer app working.
