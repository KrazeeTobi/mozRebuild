Windows Dial-Up Product Config Files

The Dial-Up Products use several configuration files for holding the various settings that are used by Dial-Up Edition. They are detailed in the below, with target links to sections that show the standard settings.

Definitions:

  • "New accounts" -- Account Setup screens to create a new account, real-time, through an ISP's account server.
  • "Existing accounts" -- Account Setup screens to manually setup an account the user already has, or to manually setup an account if the ISP doesn't have an account server.
3.0 Files 4.0 Equivalents Description
reg.ini register.htm For new accounts, holds all of the name-value pairs that are sent up to an account server CGI script at the start of account creation.
Regserv.sr Sample.IAS  For new accounts, holds the information for connecting to the ISP's account server, including 800#, login name, password, DNS addresses, and URL. In 4.0, multiple IAS files can exist, with any name, and the user will be shown a drop-down list from which to choose.
ISP.sr Sample.NCI For existing accounts, holds the information for pre-configured ISP accounts, such as mail and news hosts and DNS addresses. Commonly used when ISPs will not be offering real-time sign-up through an account server. In 3.0, the ISP.SR file could have any name. In 4.0, multiple NCI files can exist, with any name, and the user will be shown a drop-down list from which to choose.
Acctset.ini Acctset.ini For new and existing accounts, holds the Yes-No flags for which Account Setup screens should be shown. Also holds naming strings and various other settings.
Setup.ini Setup.ini Installer file that sets product names on InstallShield screens.
