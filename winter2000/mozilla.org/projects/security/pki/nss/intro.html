Introduction to Network Security Services

Newsgroup: netscape.public.mozilla.crypto
Writer: Sean Cotter
Yell at the manager: Bob Lord

Network Security Services (NSS) is a set of libraries designed to support cross-platform development of communications applications. Applications built with the NSS libraries support the Secure Sockets Layer (SSL) protocol for authentication, tamper detection, and encryption and the PKCS #11 protocol for cryptographic token interfaces.

Overview of NSS Libraries

Network Security Services consists of interlocking APIs in several libraries. Figure 1 provides a high-level view of the relationships among these core libraries:

  • ssl.lib provides an API for core SSL operations.

  • cert.lib provides an API for certificate operations. In Figure 1, "High Cert" represents those cert.lib interfaces that are directly accessible to both SSL and to an application, and "Low Cert" represents those cert.lib interfaces that are accessible only via PKCS #11 interfaces (that is, via secmod.lib APIs).

  • key.lib provides an API for key operations. In Figure 1, "High Key" represents those key.lib interfaces that are directly accessible to both SSL and to an application, and "Low Key" represents those key.lib interfaces that are accessible only via PKCS #11 interfaces (that is, via secmod.lib APIs).

  • secmod.lib provides an API for various operations involving PKCS #11.

  • dbm.lib supports the database operations performed by the cert.lib and key.lib APIs.

  • nspr.lib provides an API for operations involving the Netscape Portable Runtime (NSPR), which provides low-level cross platform support for operations such as threading and I/O. (Note that NSPR is a separate Mozilla project; see Netscape Portable Runtime for details.)

Figure 1    Relationships among core NSS libraries

In addition to the core libraries shown in Figure 1, NSS makes use of the following libraries for additional operations:

  • crypto.lib provides an API for basic cryptographic operations.

  • hash.lib provides an API for hashing operations.

  • pkcs12.lib provides an API for PKCS #12 operations--that is, operations related to exporting and importing certificates and keys.

  • pkcs7.lib provides an API for PKCS #7 operations--that is, for operations related to digital signatures and encryption.

  • secutil.lib provides an API for additional cryptographic operations.
The NSS documentation set documents the core APIs provided by these libraries that allow developers to support SSL in their applications.

NSS Documentation Set

The documentation set for Network Security Services currently consists of the following documents:

  • Introduction to Network Security Services (this document). Overview of the libraries and documentation.

  • SSL Reference. Describes the SSL API and provides sample code that demonstrates its use.

  • Using the Certificate Database Tool. Guide to the Certificate Database Tool (certutil), used to manipulate the certificate and key databases.

  • Using the Security Module Database Tool. Guide to the Security Module Database Tool (secmod), used to manipulate the security module database.

  • Using the SSL Debugging Tool. Guide to the SSL Debugging Tool (ssltap), used to debug SSL applications.
As mentioned above, NSS is built on top of NSPR. The API documentation for NSPR is available at NSPR Reference.

What You Should Already Know

The NSS documentation assumes that you are familiar with the following programming topics:

  • concepts and techniques of public-key cryptography

  • the Secure Sockets Layer (SSL) protocol

  • the PKCS #11 standard for cryptographic token interfaces

  • cross-platform development issues and techniques
