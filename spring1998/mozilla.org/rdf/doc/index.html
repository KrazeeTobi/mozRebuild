RDF in Mozilla

Guha
Robert Churchill
John Giannandrea


The Resource Description Framework (RDF) is used in Mozilla for many purposes. Its a swiss army knife and we will use it wherever it makes sense to use the RDF data model as a representation language. In particular, it is currently being used as the foundation for NavCenter/Aurora.

The relationship between NavCntr/Aurora, the HyperTree abstraction layer and RDF is depicted in this image.

NavCntr is a single UI widget for managing bookmarks, history, search results, file systems, ftp, sitemaps etc. All these "data types" are reflected into the RDF data model. Not just the data in the Aurora view, but even information such as the commands applicable to an object and the columns that make sense for it are all reflected into the RDF graph.

It is our goal that the core RDF module will not have any dependencies on the rest of Mozilla (other than NSPR and JavaScript), so you will be able to take the RDF module and use it elsewhere.

Note: RDF is in the process of being standardized by the W3C. You can take a look at the public drafts that are accessible here. We expect Mozilla's RDF implementation to track the evolving RDF standard as specified in future W3C recommendations. Aspects of the current draft could change but we expect that the core data model that we rely on is unlikely to change in a fashion that is incompatible with our implementation.
