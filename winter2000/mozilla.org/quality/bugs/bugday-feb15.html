BugDay - February 15, 2000
Page created by Jan Leger

Hi All!!

Need help today verifying Resolved/Fixed bugs.  Most QA engineers like to verify their own bugs, but I have located several where anyone can help.

Resolved/Fixed with No QA Contact
Please review bugs in this section, then do the following:

  1. Assign bug to yourself as QA contact
  2. Verify bug no longer occurs with latest daily build
  3. If bug no longer occurs, indicate as such within the bug making sure to note build tested against, move Resolution field to Verified.
  4. If bug still occurs, comment build tested and results in the bug, reopen the bug

Resolved/Fixed with QA Contact as leger or cbegle
cbegle and leger are defaults on the QA Contact field for new Browser General bugs.  We try to keep up and move bugs to correct folks, but sometimes don't have a chance to get to them before they get fixed.  To help with these, click on queries below and:
  1. Assign bug to yourself as QA contact
  2. Verify bug no longer occurs with latest daily build
  3. If bug no longer occurs, indicate as such within the bug making sure to note build tested against, move Resolution field to Verified.
  4. If bug still occurs, comment build tested and results in the bug, reopen the bug
