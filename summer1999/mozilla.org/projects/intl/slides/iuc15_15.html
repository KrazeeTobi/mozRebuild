"XUL + Language-specific DTD"
Sample XUL:  openLocation.xul  
 <window ...  title="&title.label;" > 

<html:div style="width: 100%;"> &prompt1.label;</html:div>
<html:div style="width: 100%; margin-bottom: 0.5em;">&prompt2.label;</html:div>
...
 </window>
 

 Sample DTD: toolbar.dtd
  <!ENTITY title.label "Open File or Location">
<!ENTITY prompt1.label "Enter the World Wide Web location (URL) you would like to open or">
<!ENTITY prompt2.label "use the Choose File... button to select a local file:">
 

 
One obvious advantage of the DTD solution is that it is an application of the standard feature recommended in XML specification 1.0. It’s easy to convince the XML parser developers to support this feature. For the same reason, XUL writers are more likely to adopt this strategy due to its standard compliance.
